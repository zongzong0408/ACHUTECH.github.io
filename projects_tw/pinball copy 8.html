以 ggb5 模擬彈珠檯並分析落點
BLOCK
using GeoGebra Classic Version 5
BLOCK
動機
    因為對在夜市出現的彈珠機台感到興趣，也常看到若彈珠進了某幾格可以得到獎品。因此若能模擬、了解彈珠的落點分布，也許就能以期望值計算大方向的賺賠。
BLOCK
目的
    分析彈珠下落點位置的分布情形，看彈珠最有機會出現在哪一格。
BLOCK
實驗方法
     ▪ 利用內建試算表、Sequence() 製作彈珠檯框架和障礙物
     ▪ 利用滑桿製作開始模擬的開關，和彈珠的移動(把滑桿想成迴圈)
     ▪ 在滑桿內設置程式，讓彈珠隨機往左或是往右(碰到邊界時需要轉向不會碰到邊界的那一邊，底部除外)
     ▪ 在滑桿內設置程式，讓彈珠碰到底部隱藏細線時動態繪製統計長條圖並回到初始最高點
BLOCK
遇到的困難
    無法以古典力學方式模擬彈珠落下軌跡和落點。
BLOCK
解決辦法
    讓彈珠以亂數選擇轉向，並碰到左右兩側邊界時往不會碰到邊界的方向移動(使用向量移動顯示移動軌跡)。
BLOCK
結論
    在模型製作之前假設彈珠落點為二項分布圖形，不過模型結果只能宣告其為鐘型曲線分布，並不能嚴格的稱之為二項分布。
